        # redis — RESP-протокол и redis-cli

        Homework-шаблон для урока **l2_resp_and_cli** (RESP-протокол и redis-cli) на платформе Vibe Learn.

        ## Что делать

        Напиши минимальный RESP-парсер на Go: принимает байтовый поток, парсит SET/GET/INCR команды,
отвечает по RESP2. Тесты подкормят твоему серверу зафиксированные запросы и проверят
байт-в-байт ответы.

## Контекст (из transfer-задачи урока)

На проде проблема: один из бэкендов раз в минуту дёргает `redis-cli KEYS 'session:*'`
для дашборда «активные сессии». Сейчас keyspace ~3M ключей. Latency p99 на других
клиентах подскакивает с 1мс до 400мс **ровно каждую минуту**, синхронно с обновлением
дашборда, и пользователи жалуются на периодические таймауты.

**Вопрос:** объясни механику проблемы и предложи минимум два способа починки,
не отказываясь от самой метрики. Для каждого способа укажи trade-off.
Бонус: как ты бы доказал, что виноват именно KEYS, имея доступ только к серверу?

## Recap из урока

- RESP — простой текстово-бинарный протокол с 5 маркерами: `+`/`-`/`:`/`$`/`*`.
- Команда от клиента — всегда **массив bulk-string'ов**: `*3\r\n$3\r\nSET\r\n$3\r\nkey\r\n$3\r\nval\r\n`.
- Любая O(N) команда (`KEYS *`, `LRANGE 0 -1`) на большой структуре = заморозка single-thread'а на секунды. Используй `SCAN`, `LRANGE` с границами, итерируй.
- `Pipeline` экономит сетевой RTT (1000 команд за один пакет), но **не даёт атомарности** — для этого `MULTI / EXEC` или Lua.
- `redis-cli` — швейцарский нож: `--scan`, `--bigkeys`, `--latency`, `--stat`, `monitor` (последний — коротко!).

        ## Как работать

        1. Платформа Vibe Learn создаёт копию этого репо в твоём GitHub-аккаунте по клику «Начать домашку» на странице урока (через GitHub `/generate`, codecrafters-pattern).
        2. Склонируй копию локально, реализуй TODO в `main.go`, прогони тесты, запушь.
        3. CI (`.github/workflows/ci.yml`) запускает `go vet` + `go test ./...` на каждый push. Платформа слушает результат через webhook от GitHub Actions и обновляет статус домашки на странице урока.

        ## Локальное окружение

        - Go 1.22+
        - Docker + docker-compose — `docker compose up -d` поднимает single-node Redis 7 на `localhost:6379` (с включёнными keyspace-notifications и AOF). Адрес переопределяется через env `REDIS_ADDR`.

        ## Запуск

        ```bash
        # Поднять локальный Redis
        docker compose up -d

        # Прогнать тесты (интеграционный включается через REDIS_INTEGRATION=1)
        go test ./...
        REDIS_INTEGRATION=1 go test ./...

        # Запустить main (печатает marker; замени stub на реализацию)
        go run .
        ```
